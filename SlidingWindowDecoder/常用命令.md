```bash

```


## 这个项目在做什么

这个项目是论文 *Toward Low-latency Iterative Decoding of QLDPC Codes Under Circuit-Level Noise* 的代码。核心目标是研究量子 LDPC 码在 circuit-level noise 下的低延迟解码，尤其是把大的时空 syndrome 解码问题切成多个滑动窗口，然后在每个窗口里使用 BP+OSD 或 GDG 解码。

大致流程是：

1. 构造量子码，例如 bivariate bicycle code、surface code、SHYPS code。
2. 用 Stim 构造带噪声的 syndrome extraction circuit。
3. 从 Stim detector error model 转成校验矩阵、逻辑观测矩阵和每个 fault 的 prior。
4. 按时间顺序切出滑动窗口。
5. 在每个窗口内解码，并把已提交区域的估计错误反馈给后续窗口。
6. 统计 flagged error、logical error、logical error per round 等指标。

## 主要代码文件作用

- `osd.py`：脚本版滑动窗口 BP+OSD 示例。默认会构造 BB code 的 Stim 电路，切窗口，用 `ldpc.BpOsdDecoder` 或本地 `osd_window` 解码。
- `guessing.py`：脚本版滑动窗口 GDG 示例。使用本地 Cython/C++ 的 `bpgdg_decoder`，默认采样更多，运行更重。
- `setup.py`：编译本地 Cython/C++ 扩展，包括 `mod2sparse`、`c_util`、`bp_guessing_decoder`、`osd_window`、`bp4_osd`。
- `src/codes_q.py`：量子 CSS 码构造工具。包含 `css_code` 类，以及 generalized bicycle、bivariate bicycle、hypergraph product、surface code 等构造。
- `src/build_circuit.py`：为 BB codes 构造 Stim syndrome extraction circuit，并把 Stim detector error model 转成解码用的 check matrix、observable matrix、prior。
- `src/build_SHYPS_circuit.py`：构造 SHYPS 相关的 Stim 电路和 detector/observable 结构。
- `src/simulation.py`：数据比特噪声模型下的仿真框架，用于比较 OSD、GDG 和额外 decoder 的逻辑错误率。
- `src/utils.py`：二元线性代数、alist、行化简、rank、kernel、inverse、图相关工具等通用函数。
- `src/__init__.py`：把本地扩展里的 decoder 暴露成 `from src import ...` 的接口。
- `src/bp_guessing_decoder.pyx`：GDG/BPGD 的 Python 绑定和单线程相关接口，编译后生成 `bp_guessing_decoder` 扩展。
- `src/osd_window.pyx`：窗口内 OSD/shortening 相关的本地加速实现。
- `src/bp4_osd.pyx`：BP4+OSD 相关本地实现。
- `src/c_util.pyx`：Cython 辅助工具函数。
- `src/mod2sparse.pyx`：把稀疏二元矩阵底层 C 实现暴露给 Python。
- `src/include/bpgd.cpp` 和 `src/include/bpgd.hpp`：GDG/BPGD 的核心 C++ 实现，包括多线程 ensemble、decimation、BP 迭代、路径搜索等逻辑。
- `src/include/mod2sparse.c` 和 `src/include/mod2sparse.h`：底层二元稀疏矩阵结构和操作。
- `src/include/mod2sparse_extra.cpp` 和 `src/include/mod2sparse_extra.hpp`：稀疏矩阵额外工具，供窗口 OSD/BP4+OSD 使用。

## 推荐阅读顺序

1. 先看 `README.md` 和 `FAQ.md`。
2. 跑通 `osd.py`，理解最小脚本入口。
3. 打开 `Round Analysis.ipynb`，看窗口是怎么切的。
4. 打开 `Sliding Window OSD.ipynb`，看完整 BP+OSD pipeline。
5. 再看 `guessing.py` 或 `Sliding Window GDG.ipynb`，理解 GDG 如何替换窗口内 decoder。
